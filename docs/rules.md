# プロジェクトルール定義

このドキュメントは、Antigravity プロジェクトの開発ルールをまとめたものです。

## 1. コミュニケーション方針
* **使用言語**: すべて日本語。
* **回答スタイル**: 簡潔かつ的確に。
* **確認フロー**: 大きな変更前には概要を提示し承諾を得る。

## 2. ディレクトリ構造・命名規則
| カテゴリ | 命名規則 / 配置 |
| :--- | :--- |
| **コンポーネント** | PascalCase / `/src/components` |
| **ディレクトリ** | kebab-case / 新規プロジェクトはサブフォルダ化 |
| **関数・変数** | camelCase |
| **定数** | UPPER_SNAKE_CASE |
| **型定義** | PascalCase / `/src/types` |
| **カスタムフック** | camelCase / `/src/hooks` |

## 3. 開発運用ルール
* **TypeScriptの徹底**: `any` 禁止。`/src/types` の型を優先利用。
* **ドキュメントの同期**: 変更時は `/docs/` も更新。
* **コードコメント**: 日本語 JSDoc。
* **AI自動化**: ファイル冒頭に役割コメント。
